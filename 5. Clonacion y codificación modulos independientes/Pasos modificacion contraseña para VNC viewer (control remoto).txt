Una vez tengamos el nuevo dispositivo reconfigurado en el código raspberry
Y funcione de manera independiente con su propio ID (En este caso poder visualizar cada ID correspondiente en la terminal de cada raspberry (en uno ID RPI-01 EN OTRO RPI-02),  enlazado a azure y voluta con sus respectivos ID diferentes, para poder visualizarlo en el VNC server y tener control remoto desde un computador
se debe cambiar su contraseña de acceso al VNC server para que no choque con el acceso a la visualización de otros dispositivos ya creados con la misma contraseña pues esta también quedará clonada.

Para esto debemos acceder a TERMINAL RASPBERRY UBICADO EN ESQUINA SUPERIOR IZQUIERDA (CTRL+ALT+T) DENTRO DEL RASPBERRY

1. Escribir comando vncserver
2. vncpasswd
3. en caso de ocurrir algun error, escribir comando sudo passwd 

inicialmente pide la contraseña actual, si no tiene asignarle una, luego pide ingresar nuev contraseña, luego se confirma se acepta con enter 
Para finalizar se escribe sudo reboot para reiniciar y guardar cambios (Hay que probar en vnc server su funcionamiento id que por defecto es pi y clave de vnc server creada en terminal raspberry)
